# BookingApp Mini (PostgreSQL + Tkinter)

Мини-система бронирования столиков с:
- PostgreSQL-драйвером (`postgres_driver.py`);
- бизнес-логикой CRUD (`backend.py`);
- графическим интерфейсом на `tkinter` (`main.py`).

## Что умеет проект

- CRUD для таблиц:
  - `users`
  - `tables`
  - `bookings`
- Автосоздание таблиц по SQLAlchemy-моделям:
  - `User`
  - `Table`
  - `Booking`
- Проверка доступности столика:
  - `check_table_availability(...)`
  - используется при `create_booking(...)` и `update_booking(...)`
- GUI на вкладках:
  - `Setup`, `Users`, `Tables`, `Bookings`, `Availability`

## Стек

- Python 3.12+
- PostgreSQL 14+
- `psycopg2-binary`
- `python-dotenv`
- `SQLAlchemy`
- `tkinter` (обычно встроен в Python)

## Последовательность запуска для пользователей

### 1) Подготовить PostgreSQL

1. Создайте базу данных (например, `booking`).
2. Создайте/используйте пользователя БД (например, `admin`).
3. Выдайте права на схему `public` (иначе таблицы не создадутся):

```sql
GRANT USAGE, CREATE ON SCHEMA public TO admin;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO admin;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO admin;
```

Если PostgreSQL дополнительно блокирует создание объектов:

```sql
ALTER DATABASE booking OWNER TO admin;
ALTER SCHEMA public OWNER TO admin;
```

### 2) Настроить переменные окружения

Создайте файл `.env` в корне проекта (рядом с `postgres_driver.py`):

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=booking
DB_USER=admin
DB_PASSWORD=your_password
```

### 3) Создать и активировать виртуальное окружение

```powershell
python -m venv venv
venv\Scripts\Activate.ps1
```

### 4) Установить зависимости

Если `requirements.txt` не содержит все нужные пакеты, установите напрямую:

```powershell
pip install psycopg2-binary python-dotenv SQLAlchemy
```

### 5) Запустить GUI

```powershell
python main.py
```

### 6) Первый запуск в интерфейсе

1. Откройте вкладку `Setup`.
2. Нажмите `Create DB tables`.
3. Далее работайте через вкладки `Users`, `Tables`, `Bookings`, `Availability`.

## Архитектура

### `postgres_driver.py`

Низкоуровневый драйвер:
- подключение к PostgreSQL;
- выполнение SQL;
- `insert/read_one/read_many/update/delete`;
- генерация `CREATE TABLE` по SQLAlchemy-модели (`create_table_from_model`).

Особенности SQL-генерации:
- PK `Integer` + autoincrement -> `GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`;
- корректные внешние ключи: `REFERENCES "table"("id")`.

### `backend.py`

Высокоуровневые функции:
- `create_tables()`
- `create_user/read_users/update_user/delete_user`
- `create_table_record/read_tables/update_table_record/delete_table_record`
- `create_booking/read_bookings/update_booking/delete_booking`
- `check_table_availability(...)`

Логика бронирований:
- при создании/обновлении брони проверяется конфликт по времени;
- если столик занят, выбрасывается `ValueError`.

### `main.py`

Tkinter-приложение:
- отдельные вкладки по сущностям;
- формы для CRUD;
- вывод результатов в JSON;
- отображение ошибок через диалоги.

## Пример использования backend без GUI

```python
from datetime import datetime
from backend import (
    create_tables,
    create_user,
    create_table_record,
    create_booking,
    read_bookings,
)

create_tables()

user = create_user(
    {
        "username": "alice",
        "email": "alice@example.com",
        "hashed_password": "hash",
        "full_name": "Alice",
        "is_active": 1,
    },
    returning=["id"],
)

table = create_table_record(
    {"table_number": 1, "seats": 4, "location": "window", "is_active": 1},
    returning=["id"],
)

create_booking(
    {
        "user_id": user["id"],
        "table_id": table["id"],
        "booking_time": datetime(2026, 3, 1, 19, 0, 0),
        "duration_minutes": 120,
    },
    returning=["id", "booking_time"],
)

print(read_bookings(limit=10))
```

## Типичные ошибки и решения

- **`ModuleNotFoundError`**  
  Установите зависимости из раздела установки.

- **`нет доступа к схеме public` (код 42501)**  
  Выдайте права на схему `public` пользователю БД.

- **`schema "users" does not exist`**  
  Проверьте, что используете актуальный `postgres_driver.py` с корректной генерацией `REFERENCES`.

- **Ошибка формата даты**  
  Для GUI используйте формат: `YYYY-MM-DD HH:MM:SS`.
